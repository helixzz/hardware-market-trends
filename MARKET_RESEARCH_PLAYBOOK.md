# 硬件市场调研执行手册

这个任务的目标，不再只是“生成一份日报”，而是每天形成可追踪、可验证、可行动的市场判断。

## 一、任务目标

每日任务必须同时完成以下四件事：

1. **采集市场证据**：尽可能扩大企业级 DDR5 RDIMM 与企业级 NVMe SSD 的可见样本覆盖。
2. **形成趋势判断**：结合最近 7-14 天历史数据，识别上涨、下跌、持平、波动放大、样本塌缩。
3. **给出采购建议**：不是泛泛而谈，而是明确指出“建议立即买 / 可等待 / 分批买 / 仅应急采购”，并说明理由。
4. **记录缺口与后续方向**：如果某类仍缺样本，说明缺的是来源、解析、还是市场公开信息本身。

## 二、每日执行顺序

### 阶段 A：读取历史上下文

在调研当日市场之前，先读取：

- 最近 3 份 `daily-YYYY-MM-DD.md`
- `tracking-table.md`
- `analysis/market-history-summary.json`
- 最近 3 份 `raw/YYYY-MM-DD/quotes.json`

目的：
- 识别哪些容量位长期没有样本
- 识别哪些价格连续几天完全不变，可能是来源过窄
- 识别哪些类别最近出现明显拐点

### 阶段 B：建立当日来源矩阵

每个类别都要按“主源 + 备源 + 校验源”组织：

- **DDR5-6400 / DDR5-5600 RDIMM**
- **PCIe4 TLC**
- **PCIe5 TLC**
- **PCIe4 QLC**

对每个类别至少说明：
- 今天实际查看了哪些来源
- 哪些来源给出公开价格
- 哪些来源只有规格页或 RFQ
- 哪些来源页面失效/结构变化/抓取失败

### 阶段 C：提取有效证据

对每条证据都判断：

- 是否企业级
- 是否口径匹配
- 是否公开可见价格
- 是否明显异常价
- 是否与历史主流价格严重偏离

不符合口径的证据可以记录在日报旁证区，但不能写入 tracking-table 主值。

### 阶段 D：生成趋势与采购判断

你必须从“今天 vs 历史”中给出判断，至少包含：

- **价格趋势**：上涨 / 下跌 / 持平 / 无法判断
- **样本质量**：改善 / 恶化 / 持平
- **采购建议**：
  - 建议立即采购
  - 建议等待
  - 建议分批采购
  - 仅应急采购
  - 暂无足够依据
- **采购理由**：
  - 价格是否处于近期区间高位/低位
  - 样本是否足够可信
  - 是否存在供给收缩风险
  - 是否只是来源塌缩导致的假信号

### 阶段 E：落盘与回报

最终必须产出：

- `daily-YYYY-MM-DD.md`
- 更新后的 `tracking-table.md`
- 当日分析快照：`analysis/daily-analysis-YYYY-MM-DD.json`
- 更新后的滚动摘要：`analysis/market-history-summary.json`

Telegram 摘要必须包含：
- 今日各类别覆盖率
- 今日新增来源
- 相对昨日最重要变化
- 采购建议（如果有）
- 仍为空的关键容量位

## 三、缺失原因分类（必须严格区分）

当某个容量位没有主值时，只能属于以下几类之一：

- `no_public_sample`：查过多源后仍无公开价格
- `parser_gap`：页面有价格，但现有自动化无法可靠提取
- `out_of_scope`：有价格，但口径不符
- `outlier_only`：有价格，但都偏离主流锚点过大
- `stale_repeated_sample`：只有陈旧/重复样本，缺乏新的市场信息

禁止把这些情况都写成“样本不足”。

## 四、采购建议规则

采购建议不是空想，必须参考以下维度：

1. 最近 7-14 天价格位置
2. 最近 7-14 天样本覆盖变化
3. 是否出现来源收缩
4. 是否出现单日异常价扰动
5. 是否存在明显的高价 OEM / 低价清仓样本混入

### 输出口径

- **建议立即采购**：价格在近期低位，且样本可信、来源分散，短期没有更好等待理由
- **建议等待**：价格处于近期高位或信号分裂，当前没有紧迫理由下单
- **建议分批采购**：价格不确定但供给风险在上升，适合拆分执行
- **仅应急采购**：价格偏高、覆盖弱、趋势不明，不适合常规下单
- **暂无足够依据**：样本质量太差，不应强行给建议

## 五、对脚本的使用原则

现有脚本与生成器只是辅助工具，不是研究本身。你可以用它们：

- 生成结构化输出
- 更新 tracking-table
- 保存原始样本

但不能因为脚本输出是 `NA`，就认定市场真的没有数据。

## 六、停止条件

满足以下任一条件后可以停止扩展来源：

- 已达到该类别“可接受覆盖率”
- 连续新增来源两轮都没有带来有效新样本
- 该类别明确只剩 RFQ/规格页，无公开价格
- 时间预算已接近上限，但已有可解释结论

如果停止，必须说明为什么停止。
